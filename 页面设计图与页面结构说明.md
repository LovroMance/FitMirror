# FitMirror 移动端PWA UI原型设计说明

请为FitMirror健身移动端PWA应用设计一套具有独特品牌感、极具吸引力的高保真UI原型，要求如下：

## 1. 页面列表（每页需有清晰布局与交互，全部基于移动端PWA场景）

- 首页（计划生成入口、打卡状态、推荐区）
- 登录页（邮箱/用户名登录、第三方登录入口、忘记密码）
- 注册页（邮箱/用户名注册、密码校验、协议勾选）
- 训练计划详情页（动作列表、计划说明、开始训练按钮）
- 训练中页面（摄像头画面、AI姿态叠加、实时动作反馈、进度条、暂停/结束）
- 训练记录日历页（GitHub风格热力图、训练详情弹窗）
- 动作库页（搜索/筛选、动作卡片、动作详情弹窗）
- 饮食推荐页（AI推荐食谱、食谱卡片）
- 个人中心页（头像、昵称、邮箱、训练统计、偏好设置、账号安全、登出）
- 404/错误页（友好提示、返回首页按钮）

## 2. 设计风格要求

- 拒绝AI风格2D深浅阴影卡片、拟物化玻璃质感等流行AI UI套路
- 追求极简、大气、留白充足、信息分区清晰
- 色调协调，主色建议采用健康、自然、活力感强的色系（如柔和蓝绿、明快橙、健康浅灰），避免高饱和和度和廉价感
- 视觉重心突出健身主题，可用大幅人物剪影、动态线条、健康运动元素点缀
- 图标、插画风格统一，线性或极简面性，避免过度装饰
- 按钮、输入框、卡片等控件采用极简无阴影或极浅阴影，圆角适中，边框清晰
- 所有页面和交互均以移动端为主，适配主流手机屏幕，去除PC端/响应式/多端适配要求
- 页面整体风格要让用户一眼感受到“专业、健康、值得信赖”，并愿意停留

## 3. 交互与体验细节

- 所有表单需有输入校验与错误提示，交互流畅
- 交互按钮需有active/disabled等状态，动效克制
- 训练中页面摄像头区域自适应，移动端全屏显示
- 日历热力图支持点击查看每日训练详情
- 个人中心支持头像上传、信息修改、密码修改、主题切换
- 支持PWA“添加到主屏幕”提示（移动端底部悬浮）
- 未登录用户访问受限页面自动跳转登录
- 所有页面均有返回首页/导航能力

## 4. 品牌氛围建议

- 以“AI赋能健康生活”为理念，强调陪伴、激励、科学训练
- 可适当融入品牌Slogan、健身激励语、成就徽章等元素
- 页面视觉统一，风格鲜明，避免“千篇一律AI产品感”

请根据以上要求，生成一套极具品牌辨识度、健身氛围浓厚、功能健全、体验流畅、视觉统一的页面原型和交互流程。
