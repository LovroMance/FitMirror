# FitMirror 架构设计与技术方案（合并版）

文档版本：v1.0
最后更新：2026年3月

---

## 一、整体架构概览

FitMirror 采用前后端分离的 Web 架构，前端专为移动端PWA（渐进式Web应用）设计，基于 Vue3 + Vite + TypeScript，集成 AI 动作识别与个性化推荐。后端（BFF）提供用户、训练、数据等 API 支持，数据存储采用 MySQL，仅存账号体系必要信息，所有隐私数据仅前端本地存储。

```
┌────────────┐      ┌──────────────┐      ┌──────────────┐
│  用户移动端浏览器 │<--->│  前端移动端PWA │<--->│   BFF服务    │
└────────────┘      └──────────────┘      └──────────────┘
         │                │                        │
         │        Dexie.js/IndexedDB              │
         │        (本地存储/AI推理)               │
         │                │                        │
         │        TensorFlow.js + PoseNet         │
         │        (本地摄像头推理)                │
         │                │                        │
         │        ECharts/日历热图                │
         │                │                        │
         │        Axios 调用 BFF API              │
         │                │                        │
         │        仅账号等必要信息存后端           │
```

---

## 二、前端架构与技术选型（移动端PWA专用）

- 框架：Vue 3 + Vite + TypeScript
- 状态管理：Pinia
- UI 组件库：Element Plus
- 路由：Vue Router
- 网络请求：Axios
- 本地存储：Dexie.js（高效易用的 IndexedDB 封装库，https://dexie.org/）
- AI 动作识别：TensorFlow.js + @tensorflow-models/posenet
- 数据可视化：ECharts（Apache ECharts）
- 训练日历热图：vue-calendar-heatmap（https://github.com/WildCodeSchool/vue-calendar-heatmap）
- PWA能力：manifest.json、service worker、推送、离线等
- 日期处理：day.js（轻量日期库，https://day.js.org/）
- Web Speech API：浏览器原生，无需第三方库
- WebSocket 支持：socket.io-client（如需实时社交激励）
- 文件导出：file-saver、xlsx（如需导出Excel）
- 代码规范：ESLint、Prettier、Stylelint、Husky、lint-staged
- 单元测试：Vitest

### 前端关键模块

| 模块       | 说明                      |
| ---------- | ------------------------- |
| 计划生成   | AI 生成个性化训练计划     |
| 动作识别   | 摄像头捕捉+PoseNet推理    |
| 实时纠正   | 动作分析、语音/文本反馈   |
| 训练记录   | 本地存储/云同步           |
| 数据可视化 | ECharts/Chart.js 展示趋势 |
| 用户管理   | 登录、注册、隐私设置      |

## 三、BFF/后端架构与技术选型

- Node.js（推荐 TypeScript 支持）
- Web 框架：Express.js
- API 聚合/转发：axios（服务端聚合远程API）
- 鉴权：jsonwebtoken（如需简单JWT鉴权）
- 部署：Vercel、Netlify、Serverless Framework（如需无服务器）
- 仅存储账号、登录凭证、基础配置
- 不存储训练计划、训练记录、动作分析等隐私数据
- 数据库选型为MySQL，结构极简
- 仅暴露最小必要RESTful API
- 可扩展第三方API聚合、云同步、AI服务等

---

## 四、数据库与本地数据

- 本地数据：IndexedDB（推荐用 Dexie.js 封装库）
- 动作库/饮食库：静态 JSON 文件，托管于 GitHub，前端定期拉取
- 云端数据库（后续）：MySQL

---

## 五、AI/ML 能力

- TensorFlow.js：前端AI推理
- @tensorflow-models/posenet：人体关键点检测
- OpenAI API（如需AI文本生成，后续可选）

---

## 六、代码规范与CI/CD

- 代码风格/质量：ESLint、Prettier、Stylelint（强制执行代码规范，建议CI中集成）
- 单元测试：Vitest 或 Jest
- CI/CD：GitHub Actions
- Git提交校验：Husky、lint-staged

---

## 七、数据流与安全

- 所有训练、健康、AI分析等敏感数据仅Dexie.js本地存储，只有用户本人可见
- 账号注册/登录/找回等通过BFF与后端交互
- BFF层不做隐私数据持久化，仅做转发/聚合
- 所有API加密传输，前端明确告知数据处理方式

---

## 八、部署与运维

- 前端：Vercel/Netlify/静态服务器
- 后端：Serverless/云函数/轻量 Node.js
- 数据库：MySQL（推荐云服务如 PlanetScale、阿里云 RDS 等）
- 日志与监控：Sentry/Serverless 日志

---

## 九、第三方服务与依赖

- AI：TensorFlow.js、OpenAI API（可选）
- UI：Element Plus
- 图表：ECharts/Chart.js
- 云存储：Firebase/OSS（可选）

## 十、版本与扩展性

- 采用语义化版本管理（SemVer）
- MVP 阶段优先核心功能，后续可扩展更多 AI 模型、社交、硬件对接
- 云同步、社交、会员等功能可在BFF/后端逐步引入
- 预留本地-云端数据迁移接口
- 后端可平滑升级为微服务/Serverless架构

---

## 十一、备注与补充

- 所有涉及用户隐私的数据（训练计划、记录、动作分析等）均存储于本地 IndexedDB，只有用户本人可见，云端仅做必要的公共数据拉取（如动作库、饮食库）。
- 后端仅存储账号、登录凭证、基础配置等必要信息，不存储训练计划、训练记录、动作分析等隐私数据。
- 训练日历热图建议直接用 vue-calendar-heatmap，省去自绘热力格子的复杂度。
- BFF 层只做数据聚合与转发，不存储用户隐私数据，便于后续扩展云端能力。
