# FitMirror 数据库架构文档

文档版本：v1.0
最后更新：2026年3月

---

## 一、数据库选型与原则

- **数据库类型**：MySQL（结构化关系型数据库）
- **存储内容**：仅账号体系相关必要信息（如账号、登录凭证、基础配置等）
- **隐私策略**：所有训练计划、训练记录、动作分析等敏感数据全部前端本地存储，后端数据库不存储任何隐私数据

---

## 二、核心数据表结构（MySQL 示例）

### 1. 用户表（users）

| 字段名         | 类型         | 约束                       | 说明                   |
| -------------- | ------------ | -------------------------- | ---------------------- |
| id             | INT          | PRIMARY KEY AUTO_INCREMENT | 用户主键               |
| username       | VARCHAR(64)  | UNIQUE NOT NULL            | 用户名/昵称            |
| email          | VARCHAR(128) | UNIQUE                     | 邮箱（可选）           |
| password_hash  | VARCHAR(255) | NOT NULL                   | 密码哈希               |
| avatar         | VARCHAR(255) |                            | 头像URL                |
| created_at     | DATETIME     | NOT NULL                   | 注册时间               |
| last_login     | DATETIME     |                            | 最后登录时间           |
| oauth_provider | VARCHAR(32)  |                            | 第三方登录类型（可选） |
| oauth_id       | VARCHAR(128) |                            | 第三方ID（可选）       |

### 2. 用户基础配置表（user_settings，可选）

| 字段名     | 类型         | 约束                       | 说明       |
| ---------- | ------------ | -------------------------- | ---------- |
| id         | INT          | PRIMARY KEY AUTO_INCREMENT | 主键       |
| user_id    | INT          | FOREIGN KEY                | 关联用户id |
| config_key | VARCHAR(64)  | NOT NULL                   | 配置项名称 |
| config_val | VARCHAR(255) |                            | 配置项内容 |

---

## 三、关系与说明

- 用户表为核心，仅与基础配置表一对多关联
- 不包含训练计划、训练记录、动作库等表，这些数据全部前端本地存储
- 所有表建议加 created_at/updated_at 字段

---

## 四、后续可扩展建议

- 社交（好友、分享、评论，仅限账号体系）
- 设备信息（如多端登录）
- 会员/付费信息（仅账号相关）
- 第三方登录与多平台账号绑定

---

### 备注

- 本数据库架构严格遵循“隐私数据前端本地存储，后端仅存账号体系必要信息”原则。
- 如需SQL建表语句、索引设计、数据流图等，可进一步细化。
