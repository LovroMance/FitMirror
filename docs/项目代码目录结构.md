# FitMirror 项目代码目录结构

文档版本：v2.1  
最后更新：2026-04-01

本文档只描述当前仓库中已经存在的结构，不保留未来规划目录。

## 一、根目录

```text
FitMirror/
├── backend/                 # Express + Prisma 后端
├── docs/                    # 项目文档
├── frontend/                # Vue3 + Vite 前端
├── scripts/                 # 根目录辅助脚本
├── .github/                 # GitHub 配置
├── .husky/                  # Git hooks
├── .env.example             # 环境变量示例
├── README.md
└── package.json             # 根目录统一脚本
```

## 二、前端结构

```text
frontend/
├── public/
│   ├── data/
│   │   └── exercises.json   # 静态动作库数据
│   ├── favicon.ico
│   └── manifest.json        # 当前仅具备基础 PWA 清单
├── src/
│   ├── api/                 # HTTP 请求封装
│   ├── assets/styles/       # 全局样式与主题变量
│   ├── components/
│   │   ├── common/          # 稳定复用组件
│   │   └── layout/          # 页面壳层组件
│   ├── composables/
│   │   ├── auth/
│   │   ├── exercises/
│   │   ├── home/
│   │   ├── plan/
│   │   └── workout/         # 页面级状态与流程编排
│   ├── config/
│   ├── db/                  # Dexie 数据库定义
│   ├── repositories/        # Dexie repository 层
│   ├── router/
│   ├── store/               # Pinia
│   ├── types/
│   ├── utils/               # 纯函数与转换逻辑
│   ├── views/
│   │   ├── auth/
│   │   ├── exercises/
│   │   ├── home/
│   │   ├── plan/
│   │   └── workout/         # 页面入口
│   ├── App.vue
│   └── main.ts
├── package.json
└── vitest.config.mjs
```

### 当前前端主页面

- `views/auth/`：登录、注册
- `views/home/`：首页入口
- `views/plan/`：计划生成、计划历史
- `views/workout/`：训练会话、训练记录热图
- `views/exercises/`：动作库

## 三、后端结构

```text
backend/
├── src/
│   ├── config/              # 环境变量与配置
│   ├── lib/                 # Prisma 等基础封装
│   ├── middlewares/         # 鉴权与统一错误处理
│   ├── modules/
│   │   ├── auth/            # 注册、登录、当前用户
│   │   ├── plans/           # 计划生成、流式输出、计划同步
│   │   └── workout-records/ # 训练记录同步
│   ├── types/
│   ├── utils/
│   └── app.ts               # 应用入口
├── prisma/
├── package.json
└── vitest.config.mjs
```

### 当前后端模块边界

- `auth`：账号注册、登录、JWT 登录态恢复
- `plans`：计划生成、流式输出、计划同步
- `workout-records`：训练记录同步
- 训练计划与训练记录当前仍保持“前端 Dexie 本地事实层 + 后端 MySQL 同步存储”的组合模型

## 四、脚本与测试

- 根目录 `package.json`：统一封装 `lint`、`test:frontend`、`test:backend`
- `frontend/src/**/*.test.ts`：前端单测
- `backend/src/**/*.test.ts`：后端单测
- `scripts/setup.sh`、`scripts/sync-data.js`：保留的辅助脚本

## 五、说明

- 目录判断以仓库当前代码为准，文档仅做导航
- 目录结构发生变更时，需要持续更新本文档
