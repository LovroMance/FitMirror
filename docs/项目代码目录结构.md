# FitMirror 项目代码目录结构

文档版本：v3.0  
最后更新：2026-03-31

本文档只描述当前仓库中与项目实现直接相关的目录职责。

## 一、仓库结构

```text
FitMirror/
├── backend/                     # 后端应用
├── docs/                        # 项目文档
├── frontend/                    # 前端应用
├── scripts/                     # 辅助脚本
├── UI-design/                   # 设计源文件
├── .env.example                 # 根环境变量示例
├── README.md
└── package.json
```

## 二、前端结构

```text
frontend/
├── public/
│   ├── data/
│   │   └── exercises.json       # 动作库静态数据
│   ├── favicon.ico
│   └── manifest.json            # PWA 基础配置
├── src/
│   ├── api/                     # HTTP 接口封装
│   ├── assets/styles/           # 全局样式与主题
│   ├── components/
│   │   ├── common/              # 通用组件
│   │   └── layout/              # 布局组件
│   ├── config/                  # 页面配置常量
│   ├── db/                      # Dexie 数据库定义
│   ├── repositories/            # 本地数据访问层
│   ├── router/                  # 路由与守卫
│   ├── store/                   # Pinia 状态
│   ├── types/                   # 类型定义
│   ├── utils/                   # 工具函数
│   ├── views/                   # 页面视图
│   ├── App.vue
│   └── main.ts
├── package.json
├── vite.config.ts
├── vitest.config.mjs
└── tsconfig.json
```

### 当前页面文件

- `Home.vue`
- `PlanGenerator.vue`
- `WorkoutLog.vue`
- `WorkoutSession.vue`
- `Exercises.vue`
- `Login.vue`
- `Registration.vue`

## 三、后端结构

```text
backend/
├── prisma/
│   ├── migrations/
│   └── schema.prisma            # Prisma 模型
├── src/
│   ├── config/                  # 环境变量配置
│   ├── lib/                     # Prisma client
│   ├── middlewares/             # 鉴权与错误处理
│   ├── modules/
│   │   ├── auth/                # 注册、登录、当前用户
│   │   └── plans/               # 计划生成与流式输出
│   ├── types/                   # 后端类型
│   ├── utils/                   # 响应和错误工具
│   └── app.ts                   # 应用入口
├── package.json
├── tsconfig.json
└── vitest.config.mjs
```

## 四、测试组织方式

- 前端测试当前与源文件就近放置，例如 `plans.test.ts`
- 后端测试当前与模块文件就近放置，例如 `auth.service.test.ts`
