# FitMirror 视觉风格与页面原型设计规范

文档版本：v3.0  
最后更新：2026-03-31

适用范围：FitMirror 当前移动端 PWA 页面设计。  
本文档只定义视觉与页面结构规范，不承载功能状态和未来功能规划。

## 一、设计定位

FitMirror 的界面应呈现“深色、克制、现代、可信”的移动端健身产品气质。

### 关键词

- 深色沉浸
- 轻量训练
- 专业可信
- 精品感
- 高效可执行

### 明确避免

- 玻璃拟态
- 赛博紫蓝风
- 廉价 neon 效果
- 复杂驾驶舱式布局
- 多主色竞争

## 二、色彩系统

### 品牌色

| 角色 | 色值 |
| --- | --- |
| Brand Primary | `#32D583` |
| Primary Hover | `#24C06F` |
| Primary Soft Glow | `#6EE7A8` |

### 中性色

| 角色 | 色值 |
| --- | --- |
| Background | `#0B0B0E` |
| Surface | `#16161A` |
| Surface Subtle | `#121316` |
| Surface Deep | `#050507` |
| Text Primary | `#FAFAF9` |
| Text Secondary | `#6B6B70` |
| Text Tertiary | `#4A4A50` |
| Border | `#2A2A2E` |

### 语义色

| 角色 | 色值 |
| --- | --- |
| Success | `#32D583` |
| Warning | `#F5C451` |
| Error | `#FF5A36` |
| Info | `#6EE7A8` |

### 使用规则

- 全局以深色背景为主，不使用大面积白底
- 薄荷绿只用于关键按钮、激活态和重点状态
- 语义色只用于反馈，不参与主视觉竞争

## 三、字体与排版

### 字体

- 品牌标题与页面主标题：`Fraunces`
- 功能性正文：`DM Sans`
- 中文回退：`PingFang SC` / `MiSans` / `Source Han Sans SC`

### 字号

| 层级 | 建议字号 |
| --- | --- |
| H1 | `28px` |
| H2 | `22px - 24px` |
| H3 | `18px - 20px` |
| Body | `14px - 16px` |
| Caption | `12px - 13px` |

### 排版规则

- 主标题用于建立品牌感，正文用于保证扫读效率
- 表单说明和按钮文案保持短句
- 页面避免出现长段营销文案

## 四、布局与组件

### 页面基准

- 设计宽度：`390px`
- 页面左右安全边距：`20px`
- 常用模块间距：`14px` / `20px` / `24px`
- 基础圆角：`18px`

### 卡片

- 使用深色底或深色渐变底
- 可使用细描边，不依赖厚重阴影
- 重点通过文字层级和主按钮体现

### 按钮

- 主按钮使用 `#32D583`
- 次按钮使用深色底加细描边
- 所有按钮必须区分默认、按下、禁用状态

### 输入框

- 背景使用 `#16161A`
- 边框使用 `#2A2A2E`
- 聚焦态切换到 `#32D583`
- 错误态切换到 `#FF5A36`
- 输入高度不低于 `48px`

## 五、图像与装饰

- 优先使用真实运动人物图
- 背景装饰可以使用轻量渐变、圆环和弱辉光
- 装饰不能压过信息层级

## 六、页面规范

本文档只覆盖当前产品范围内的页面。

### 首页

#### 页面目标

帮助用户快速进入训练计划生成流程。

#### 必备模块

- 品牌区
- 今日状态或欢迎信息
- 进入计划生成的主 CTA
- 快捷目标标签
- 底部导航

### 登录页

#### 页面目标

建立信任感并完成快速登录。

#### 必备模块

- 品牌标题
- 账号输入
- 密码输入
- 登录按钮
- 跳转注册入口

### 注册页

#### 页面目标

降低注册阻力并引导进入主流程。

#### 必备模块

- 邮箱
- 用户名
- 密码
- 确认密码
- 注册按钮
- 跳转登录入口

### 计划生成页

#### 页面目标

清晰展示训练计划并鼓励立即开始训练。

#### 必备模块

- 目标输入区
- 计划生成状态
- 计划卡片
- 动作列表
- 开始训练按钮

### 训练会话页

#### 页面目标

帮助用户专注完成当前训练流程。

#### 必备模块

- 当前动作名称
- 动作说明
- 组数、次数或时长信息
- 训练进度
- 暂停、继续、结束操作

#### 视觉要求

- 页面保持深色、低噪音
- 控件优先突出主流程，不制造视觉分散

### 训练记录页

#### 页面目标

让用户直观看到训练积累和当天详情。

#### 必备模块

- 热图
- 时间范围切换
- 统计概览
- 趋势摘要
- 当天详情
- 手动补录入口

#### 热图颜色梯度

| 强度 | 色值 |
| --- | --- |
| 空状态 | `#16161A` |
| 轻度 | `#244436` |
| 中度 | `#24C06F` |
| 高度 | `#32D583` |
| 最高强度 | `#6EE7A8` |

### 动作库页

#### 页面目标

帮助用户理解和筛选训练动作。

#### 必备模块

- 搜索栏
- 筛选标签
- 动作卡片列表
- 动作详情弹层

## 七、交互与动效

### 交互规则

- 表单必须有即时校验和错误提示
- 所有页面都要有清晰返回路径
- 关键操作按钮必须有即时反馈

### 动效规则

- 动效轻、快、短
- 页面切换不使用复杂 3D 效果
- 推荐使用淡入、轻缩放和状态高亮

### 建议时长

- 点击反馈：`120ms - 180ms`
- 弹层进入：`200ms - 240ms`
- 页面切换：`220ms - 280ms`
