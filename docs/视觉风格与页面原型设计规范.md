# FitMirror MVP 视觉风格与页面原型设计规范

文档版本：v2.1  
最后更新：2026-03-25  
适用范围：FitMirror MVP 移动端 PWA 高保真原型设计  
文档目的：为原型设计师提供统一、可执行的视觉与页面结构规范，用于输出品牌一致、可落地的移动端原型图。

---

## 1. 项目定位

FitMirror 是一款面向移动端的 AI 健身 PWA，核心价值不是“内容很多”，而是“像一面会反馈的智能镜子”，帮助用户快速开始训练、降低出错风险、建立持续打卡习惯。

### 关键词

- AI 私教
- 科学训练
- 陪伴感
- 轻量开始
- 专业可信
- 健康活力

### 核心用户感受

用户打开页面后的第一感受应该是：

“这是一款深色、克制、现代、可信，又带一点精品品牌感的 AI 健身产品。”

---

## 2. 整体视觉方向

### 基础要求
不要在页面出现任何手机本身自带的组件，例如移动端顶部的状态栏等

### 风格总述

采用“深色极简 + 薄荷绿高亮 + 轻奢编辑感”的视觉路线。整体以近黑色为底，用薄荷绿色作为唯一强高亮，配合克制留白、低噪音组件和少量品牌化标题，形成兼具专业、现代与精品感的移动端界面。

### 设计气质

- 深色沉浸：整体背景偏深，减少干扰，让注意力集中在关键操作和训练内容
- 极简克制：界面元素少而精，结构干净，避免复杂卡片叠加和装饰堆砌
- 精品感：标题和品牌层允许带一点编辑感和质感，而不是纯工具型界面
- 可信现代：正文、表单、操作区保持理性、清晰、可扫描

### 风格参考注入

以下风格特征已从 `pencil-new.pen` 中的登录页和注册页提炼，作为本项目后续认证页与核心页面的设计参考：

- 整体基调为深色沉浸式，背景统一接近黑色
- 高亮色只保留一支主绿色，不做多色竞争
- 页面视觉更像“现代精品品牌”而不是“传统健身工具”或“赛博 AI 产品”
- 认证页顶部允许使用品牌化标题和短句标语，增强品牌识别
- 组件样式统一偏圆角、低对比、低噪音，不依赖强阴影和重边框

### 明确避免

- 避免玻璃拟态、赛博紫蓝风、廉价 neon UI
- 避免一页里同时出现多个抢眼高亮色
- 避免把页面做成游戏面板、极客控制台或金融驾驶舱
- 避免所有标题都使用普通无个性的系统字体，导致品牌感过弱

## 3. 色彩系统

本项目必须统一使用以下色板，原型设计中不要自由更换主色调。整体基调应参考 `pencil-new.pen` 中登录页与注册页的认证风格，即“深底 + 薄荷绿高亮 + 白灰文字”。

### 3.1 品牌主色

| 角色 | 色值 | 用途说明 |
| --- | --- | --- |
| 品牌主色 Brand Primary | `#32D583` | 主按钮、链接、勾选状态、品牌块、激活态 |
| 主色加深 Primary Hover | `#24C06F` | 按钮按下态、激活状态、重点强调 |
| 主色柔光 Primary Soft Glow | `#6EE7A8` | 局部高亮、柔和能量感、弱发光边缘 |

### 3.2 辅助色

| 角色 | 色值 | 用途说明 |
| --- | --- | --- |
| 卡片底色 Surface Elevated | `#16161A` | 输入框、次按钮、登录辅助按钮、深色模块 |
| 细分割线 Divider | `#2A2A2E` | 分隔线、弱边框、结构切分 |
| 弱提示灰 Muted Gray | `#4A4A50` | 很弱的提示文案、分割信息 |

### 3.3 中性色

| 角色 | 色值 | 用途说明 |
| --- | --- | --- |
| 页面主背景 Background | `#0B0B0E` | 全局主背景 |
| 模块背景 Surface | `#16161A` | 输入框、深色按钮、次级卡片 |
| 次级区块背景 Surface Subtle | `#121316` | 较深的内层背景、二级容器 |
| 深色渐变尾色 Surface Deep | `#050507` | 沉浸式顶部背景、训练页底色 |
| 主文字 Text Primary | `#FAFAF9` | 标题、正文重点信息 |
| 次文字 Text Secondary | `#6B6B70` | 标签、说明、辅助文案 |
| 弱文字 Text Tertiary | `#4A4A50` | 占位符、分割说明、禁用态 |
| 描边 Border | `#2A2A2E` | 分隔线、弱描边、结构线 |

### 3.4 语义色

| 角色 | 色值 | 用途说明 |
| --- | --- | --- |
| Success | `#32D583` | 成功、完成、激活 |
| Warning | `#F5C451` | 中度提醒、训练姿态注意 |
| Error | `#FF5A36` | 错误、失败、异常提示 |
| Info | `#6EE7A8` | 一般信息提示、轻说明反馈 |

### 3.5 色彩使用比例建议

- 65%：深色背景与深色模块
- 25%：白色与灰色文字信息
- 10%：品牌绿色与少量语义色

### 3.6 关键要求

- 主色必须以 `#32D583` 为核心，不可替换成蓝紫色系
- 全局必须以深背景承载内容，不再以大面积白底作为主页面基调
- 高亮绿色只能用于关键按钮、激活态、重要链接和状态反馈，不能全页泛滥
- 登录页、注册页、训练页和首页都应保留一致的深色沉浸感

## 4. 字体与排版

### 字体系统

本项目采用“双字体策略”，这是风格识别的一部分，应尽量保持一致。

- 品牌标题 / 页面主标题：`Fraunces`
- 正文 / 标签 / 表单 / 按钮 / 辅助文案：`DM Sans`
- 中文兼容回退：`PingFang SC` / `MiSans` / `Source Han Sans SC`

### 使用原则

- `Fraunces` 只用于品牌名、登录注册页标题、少量高层级模块标题
- `DM Sans` 负责所有功能性文字，确保现代、干净、可读
- 不建议整站所有标题都使用 `Fraunces`，避免品牌感过度用力
- 不建议在正文区混入第三套字体

### 字重建议

- 页面主标题：`Fraunces 500`
- 模块标题：`DM Sans 600`
- 正文：`DM Sans 400`
- 字段标签：`DM Sans 500`
- 按钮文字：`DM Sans 600`

### 字号层级

| 层级 | 建议字号 | 用途 |
| --- | --- | --- |
| H1 | 28px | 登录/注册页主标题、品牌标题 |
| H2 | 22-24px | 页面标题、主模块标题 |
| H3 | 18-20px | 卡片标题、弹窗标题 |
| Body | 14-16px | 正文、表单、列表内容 |
| Caption | 12-13px | 标签、说明、状态文本 |

### 排版原则

- 标题负责品牌感，正文负责功能清晰度
- 单屏内容优先“扫读效率”，不要堆长段文本
- 标题和副标题之间要有明显呼吸感，避免挤压
- 表单和按钮文案保持短句，不做营销式长文案

## 5. 布局与组件规范

### 画板基准

- 设计基准：移动端宽度 `390px`
- 圆角基准：`18px`
- 页面左右安全边距：`20px`
- 模块间距：`14px` / `20px` / `24px`
- 底部导航高度：`72px` 左右

### 卡片风格

- 深灰黑底或深绿黑渐变底
- 细描边或无描边，允许非常轻的绿色辉光
- 圆角 `18px` 到 `28px`
- 视觉重点靠高对比文字、薄荷绿 CTA、局部渐变和状态色，而不是厚重投影

### 按钮规范

#### 主按钮

- 背景：`#32D583`
- 文字：`#0B0D0B`
- 圆角：`18px` 到 `999px`
- 状态清晰：默认 / 按下 / 禁用

#### 次按钮

- 深色底 + 细描边
- 文本和图标优先使用 `#F5F7F2`

#### 弱按钮

- 深灰底 `#16161A`
- 文本使用 `#6B6B70` 或 `#FAFAF9`

### 输入框规范

- 深色背景 `#16161A`
- 边框 `#2A2A2E`
- 聚焦态边框切换为 `#32D583`
- 错误态边框切换为 `#FF5A36`
- 输入区域尽量高于 `48px`

### 标签与徽章

- AI 标签：深色底 + 薄荷绿色描边或薄荷绿色文字
- 成就标签：薄荷绿色底 + 深色文字
- 风险提示标签：深色底 + 橙黄/红色文字

### 图标风格

- 统一使用线性或简洁实体图标
- 线条简洁，偏运动产品和状态面板风格
- 图标颜色以白色、薄荷绿、灰色为主，红点仅用于提醒

## 6. 图像与插画方向

### 图片使用建议

- 首选真实运动人物图，而非 AI 味过重的插图
- 画面尽量选择低环境噪音、暗背景、局部打光明确的运动照片
- 人物状态应体现“正在训练中”，有力量感、专注感，不要过于商业摆拍

### 背景装饰建议

- 可少量加入圆环、能量弧线、微弱辉光、进度环，表现“训练能量”和“状态激活”
- 装饰必须克制，不能抢走内容层级
- 首页 Hero 区建议使用深绿到黑色渐变，不要再使用大面积浅色 Hero

## 7. 页面级设计说明

以下页面均基于移动端 PWA 设计，所有页面应保持统一的深色基底、统一卡片体系、统一底部导航逻辑和统一的薄荷绿激活态语言。

### 7.1 首页

#### 页面目标

让用户快速理解产品价值，并立即开始生成训练计划。

#### 必备模块

- 顶部品牌区：Logo / Slogan / 今日状态
- 核心输入区：一句话描述需求的 AI 输入框
- 快捷目标标签：减脂、塑形、核心、居家无器械、10分钟训练
- 今日打卡状态卡：连续天数、完成次数、鼓励文案
- 推荐区：推荐计划 / 推荐动作 / 推荐饮食
- 底部导航

#### 视觉重点

- 首页首屏必须突出“今日挑战”或“开始训练”类高亮卡片
- 生成计划按钮或挑战 CTA 必须使用薄荷绿色，成为最强视觉焦点
- 顶部建议使用深绿到黑色渐变 Hero，并允许加入头像、通知、能量状态等组件

### 7.2 登录页

#### 页面目标

建立信任感，快速登录，不制造复杂门槛。

#### 必备模块

- 品牌标题与一句价值说明
- 用户名/邮箱输入框
- 密码输入框
- 登录按钮
- 第三方登录
- 忘记密码
- 跳转注册入口

#### 视觉重点

- 页面保持克制，但不能做成普通白底表单页
- 登录页可采用深色背景 + 独立圆角表单卡片，顶部保留品牌欢迎区
- 可搭配一句关于隐私与本地识别的短说明，增强信任

### 7.3 注册页

#### 页面目标

降低注册阻力，突出“几步即可开始训练”。

#### 必备模块

- 用户名/邮箱
- 密码与确认密码
- 协议勾选
- 注册按钮
- 已有账号去登录

#### 设计要求

- 清晰展示密码规则
- 注册成功后流程可直接引导到生成计划

### 7.4 训练计划详情页

#### 页面目标

让用户确认本次训练内容，并愿意立即开始。

#### 必备模块

- 计划标题与适用目标
- 训练时长、消耗估算、难度等级
- 动作列表卡片
- 每个动作的组数 / 时长 / 说明
- AI 备注与注意事项
- 底部固定“开始训练”按钮

#### 视觉重点

- 信息清晰比炫酷更重要
- 可使用薄荷绿色按钮、进度环、圆角标签强化计划的“可执行感”
- 动作列表建议使用整齐卡片，而不是大段文字描述

### 7.5 训练中页面

#### 页面目标

这是产品最核心页面，要体现“镜子 + AI 实时反馈”的差异化体验。

#### 必备模块

- 全屏摄像头区域
- AI 姿态识别线框叠加
- 当前动作名称
- 实时反馈提示
- 组数 / 时间 / 进度信息
- 暂停、继续、结束按钮

#### 视觉重点

- 页面采用深色背景层，例如 `#050507` 到 `#16161A`
- AI 识别线建议使用 `#32D583` 或 `#6EE7A8`
- 错误提醒用 `#F5C451` 或 `#FF5A36`
- 交互控件尽量贴边排布，避免遮挡摄像头主体

### 7.6 训练记录日历页

#### 页面目标

强化“坚持有回报”，让用户看到自己正在进步。

#### 必备模块

- 月度统计概览
- GitHub 风格热力图
- 连续打卡记录
- 某日训练详情弹窗
- 趋势概览

#### 视觉重点

- 热力图颜色梯度必须来自主色体系，并适配深底页面
- 推荐梯度如下：

| 强度 | 色值 |
| --- | --- |
| 空状态 | `#16161A` |
| 轻度 | `#244436` |
| 中度 | `#24C06F` |
| 高强度 | `#32D583` |

### 7.7 动作库页

#### 页面目标

帮助用户探索新动作，建立“训练资源库”认知。

#### 必备模块

- 搜索栏
- 筛选标签：部位、难度、器械、时长
- 动作卡片列表
- 动作详情弹窗

#### 视觉重点

- 卡片需突出动作名称、部位、难度
- 缩略图适合做成暗色照片卡片，文字直接叠加在图上
- 选中筛选项建议使用薄荷绿色描边或薄荷绿色填充

### 7.8 饮食推荐页

#### 页面目标

作为训练的轻延展模块，提供“能执行”的饮食建议，而不是复杂营养学页面。

#### 必备模块

- 今日推荐食谱
- 目标筛选：减脂、增肌、轻负担、高蛋白
- 食谱卡片
- 营养简表

#### 视觉重点

- 页面仍以深色为主，食物图和少量薄荷绿色点缀即可
- 页面保持轻松，不做医院式营养报告风格

### 7.9 个人中心页

#### 页面目标

汇总用户信息、训练成就与设置项，保持整洁可信。

#### 必备模块

- 头像、昵称、账号信息
- 连续训练天数 / 完成次数 / 收藏动作
- 偏好设置
- 账号安全
- 关于隐私与数据说明
- 退出登录

#### 视觉重点

- 成就数字可做高对比强调，允许薄荷绿色或白色大数字
- 设置列表保持规整，避免过于运营化

### 7.10 404 / 错误页

#### 页面目标

友好、轻量、不打断情绪。

#### 必备模块

- 简短说明
- 返回首页按钮
- 可选插画或简洁图形

#### 视觉重点

- 不要做成严肃报错系统页
- 保持品牌一致的温和语气，但视觉仍需维持深色体系

## 8. 交互与动效原则

### 交互原则

- 所有关键按钮必须有默认、按下、禁用状态
- 表单必须有即时校验与错误提示
- 未登录访问受限页面时自动跳转登录
- 所有页面都要有清晰返回路径

### 动效原则

- 动效轻、快、短
- 页面切换不做复杂 3D 或炫技过场
- 推荐使用淡入、轻微缩放、按钮发光或状态环填充反馈

### 建议动效时长

- 点击反馈：`120ms` 到 `180ms`
- 弹窗进入：`200ms` 到 `240ms`
- 页面切换：`220ms` 到 `280ms`

### PWA 特殊交互

- 底部悬浮“添加到主屏幕”提示卡
- 弱网/离线时应有轻提示，不制造恐慌
- 摄像头授权前需有简短说明页，强调“仅本地识别，不上传视频”

## 10. 给原型设计师的最终风格总结

请将 FitMirror 设计成一个：

“像健康生活品牌，而不是炫技 AI 工具”的移动端产品。

它应该具备以下统一印象：

- 色调以黑色/深灰黑为底，薄荷绿色为最强高亮
- 信息结构克制，不拥挤，但视觉必须更有沉浸感和力量感
- AI 感体现在反馈、激活态、能量状态和数据高亮，不体现在廉价特效
- 第一眼感觉高级和专注，第二眼想点开始训练，第三眼愿意持续回来打卡
