# FitMirror 架构设计与技术方案

文档版本：v2.0  
最后更新：2026-03-31

## 一、整体架构

FitMirror 采用前后端分离架构，当前只支持移动端 PWA 方向。

```text
移动端浏览器 / PWA
  -> Vue 3 + Vite 前端
  -> Axios 调用后端 API
  -> Dexie 持久化本地训练数据

Node.js 后端
  -> Express 路由层
  -> Prisma 访问 MySQL
  -> 仅存储账号体系数据
```

## 二、前端架构

### 技术栈

- Vue 3
- Vite
- TypeScript
- Pinia
- Vue Router
- Element Plus
- Axios
- Dexie
- day.js
- ECharts
- vue-calendar-heatmap

### 前端模块划分

- `views/`：页面级视图
- `components/`：通用和布局组件
- `api/`：接口封装
- `store/`：全局状态
- `repositories/`：本地数据访问
- `db/`：Dexie 数据库定义
- `config/`：页面配置常量
- `types/`：类型定义
- `utils/`：纯工具函数
- `assets/styles/`：全局样式与主题令牌

### 当前前端页面

- 首页 `/`
- 计划生成页 `/plans/generate`
- 训练记录页 `/records`
- 训练会话页 `/workouts/session`
- 动作库页 `/exercises`
- 登录页 `/login`
- 注册页 `/register`

## 三、后端架构

### 技术栈

- Node.js
- Express
- TypeScript
- Prisma
- MySQL
- JWT
- bcryptjs

### 后端模块划分

- `modules/auth`：注册、登录、当前用户
- `modules/plans`：训练计划生成与流式输出
- `middlewares/`：鉴权和错误处理
- `config/`：环境变量读取与校验
- `utils/`：统一响应和错误类型

### 当前 API

- `GET /api/health`
- `POST /api/auth/register`
- `POST /api/auth/login`
- `GET /api/auth/me`
- `POST /api/plans/generate`
- `GET /api/plans/generate/stream`

## 四、数据架构

### 后端数据库

- MySQL 只存账号体系数据
- 当前 Prisma 模型只有 `User`

### 前端本地数据

- `plans`
- `workout_records`
- `settings`

训练计划和训练记录当前保存在前端 Dexie，不进入后端数据库。

## 五、计划生成方案

- 前端提供稳定通道和流式通道两种调用方式
- 后端优先调用 DeepSeek
- 外部生成失败或超时后回退到模板计划
- 响应中保留计划来源标记，供前端展示

## 六、安全与边界

- 受保护页面依赖 JWT 登录态
- 后端仅处理账号体系和计划生成，不承接训练记录存储
- 当前不包含动作识别、摄像头推理和语音反馈
- 当前不包含完整离线缓存、推送和跨设备同步
