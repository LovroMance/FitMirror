# FitMirror 项目状态

文档版本：v1.14  
最后更新：2026-04-02

本文档用于记录项目当前进度、阶段结论、阻塞项和交接信息。  
本文档不承载规范定义；涉及需求、功能、架构、数据库、UI 规范时，统一以 `docs/` 下对应规范文档为准。

## 一、文档职责

### 稳定基线文档

- `docs/开发指南.md`：长期开发规范、目录约束与交付要求，原则上只在开发规则变化时修改
- `docs/功能说明.md`：产品功能范围与功能边界，原则上只在功能范围变化时修改
- `docs/视觉风格与页面原型设计规范.md`：视觉语言与页面结构规范，原则上只在设计方向变化时修改
- `docs/需求分析.md`：产品目标、目标用户、核心场景与需求边界，原则上只在产品方向变化时修改

### 持续更新文档

- `docs/数据库架构.md`：当前数据存储事实、表结构与同步边界，数据结构变化时更新
- `docs/项目代码目录结构.md`：当前仓库目录事实与模块边界，结构变化时更新
- `docs/项目状态.md`：阶段进度、结论、阻塞与交接信息，作为持续更新主文档
- `docs/文档索引.md`：文档职责、查阅顺序与维护约定，文档体系调整时更新

## 二、当前状态

- 项目当前处于移动端 PWA MVP 阶段
- 主流程已经形成可用闭环：注册 / 登录 -> 生成计划 -> 开始训练 -> 完成训练 -> 写入记录 -> 查看热图
- 文档体系已完成一轮收口，规范类内容已从进度类内容中分离
- “训练记录 -> 计划历史”的联动增强已完成首版落地，当前可从记录详情回看关联计划，并在计划历史页承接定位与使用痕迹
- 训练闭环承接体验已继续增强，记录页现在会对“刚完成训练”的结果提供顶部反馈、一键查看入口和详情高亮
- P0 主链路已完成一轮联调收口，计划同步、训练完成跳转、训练记录同步和记录页承接已验证通过
- P1 计划编辑能力已进入第二步：计划生成页支持从动作库替换单个动作，编辑草稿逻辑也已按职责拆分

## 三、当前已完成

- 账号体系：
  - 注册
  - 登录
  - 当前用户信息获取
  - 登录态恢复与路由守卫
- 训练计划：
  - 稳定通道生成
  - 流式通道生成
  - DeepSeek 失败时模板回退
  - 最近计划保存、恢复、删除
  - 当前计划轻量编辑（标题、总时长、动作顺序、删除动作）
  - 当前计划单动作替换（从动作库回填到编辑草稿）
  - 计划历史列表独立页面
  - 计划云同步
- 训练闭环：
  - 从计划页进入训练会话
  - 按动作顺序推进训练
  - 完成训练后自动写入训练记录
- 训练记录：
  - 最近六周热图
  - 周/月切换
  - 训练统计
  - 趋势摘要
  - 当天详情增强展示
  - 记录详情关联计划摘要与回看入口
  - 手动记录 / 原计划已删除 / 可回看计划 三类详情状态区分
  - 手动补录入口
- 计划历史增强：
  - 按训练记录传入的 `planId` 自动展开并定位目标计划
  - 显示计划是否已用于训练及最近使用标识
  - 按“最近使用优先，其次创建时间倒序”排序历史计划
  - 支持“全部计划 / 已用于训练 / 未用于训练”前端筛选
- 动作库：
  - 静态动作数据读取
  - 关键词、部位、难度、器械筛选
  - 动作详情弹层
  - 计划页联动预筛选
  - 收藏与最近查看

## 四、当前技术边界

- 后端当前承接账号体系、计划生成和本地数据同步
- 训练计划与训练记录事实源当前仍以前端 Dexie 为先，并同步到后端 MySQL
- 后端数据库当前包含账号、计划、训练记录模型
- 当前不把桌面端作为正式适配目标

## 六、下次继续时建议先做的事

1. 若继续推进 P1，建议先从“计划编辑第三步”继续，优先决定是否支持新增动作或只支持继续替换现有动作
2. 本地联调默认保持前端 `127.0.0.1:4173`、后端 `127.0.0.1:3000`；若 DevTools MCP 因残留 Chrome 进程无法接管浏览器，先关闭旧 Chrome 进程再重试

## 七、本次更新记录

- 已将计划编辑草稿从 `usePlanGenerator` 中拆分为职责更清晰的独立模块，并新增编辑会话存储与动作替换映射工具，计划编辑相关命名保持直观明确
- 已在计划编辑态接入“从动作库替换单个动作”能力：点击替换后会带着当前动作上下文进入动作库，选中动作后自动回到计划页回填到草稿，再可继续保存并开始训练
- 已补充前端测试，覆盖独立编辑草稿模块、替换动作回填链路和计划编辑回归场景；前端 lint、前端测试、前端 build 均已通过，并已用 Chrome DevTools MCP 验证“计划页替换动作 -> 保存编辑 -> 进入训练页”主路径，控制台无新增报错

## 八、下一步建议

1. 下一步建议优先决定是否支持“新增动作到计划”，如果做，建议继续复用当前动作库选取链路，而不是单独做一套自定义表单
2. 若计划编辑继续扩展，建议把“替换前后摘要”和“未保存变更提示”补上，提升编辑过程中的可感知反馈
