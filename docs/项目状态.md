# FitMirror 项目状态

文档版本：v1.0  
最后更新：2026-03-31

本文档用于记录项目当前进度、阶段结论、阻塞项和交接信息。  
本文档不承载规范定义；涉及需求、功能、架构、数据库、UI 规范时，统一以 `docs/` 下对应规范文档为准。

## 一、当前状态

- 项目当前处于移动端 PWA MVP 阶段
- 主流程已经形成可用闭环：注册 / 登录 -> 生成计划 -> 开始训练 -> 完成训练 -> 写入记录 -> 查看热图
- 文档体系已完成一轮收口，规范类内容已从进度类内容中分离

## 二、当前已完成

- 账号体系：
  - 注册
  - 登录
  - 当前用户信息获取
  - 登录态恢复与路由守卫
- 训练计划：
  - 稳定通道生成
  - 流式通道生成
  - DeepSeek 失败时模板回退
  - 最近计划保存、恢复、删除
  - 计划历史列表独立页面
  - 计划云同步
- 训练闭环：
  - 从计划页进入训练会话
  - 按动作顺序推进训练
  - 完成训练后自动写入训练记录
- 训练记录：
  - 最近六周热图
  - 周/月切换
  - 训练统计
  - 趋势摘要
  - 当天详情
  - 手动补录入口
- 动作库：
  - 静态动作数据读取
  - 关键词、部位、难度、器械筛选
  - 动作详情弹层
  - 计划页联动预筛选
  - 收藏与最近查看

## 三、当前未完成

- 完整 PWA 能力
- 动作识别
- 饮食推荐
- 训练记录云同步

## 四、当前技术边界

- 后端当前承接账号体系、计划生成和本地数据同步
- 训练计划与训练记录事实源当前仍以前端 Dexie 为先，并同步到后端 MySQL
- 后端数据库当前包含账号、计划、训练记录模型
- 当前不把桌面端作为正式适配目标

## 五、当前主要风险

- PWA 当前只有 `manifest.json`，不能视为完整 PWA 能力
- 计划与训练记录已具备最小云同步能力，但仍缺少更强的冲突处理和同步状态可见性
- 文档虽然已收口，但后续功能变更如果不同步文档，仍可能再次失真
