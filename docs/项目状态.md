# FitMirror 项目状态

文档版本：v1.30  
最后更新：2026-04-05

本文档用于记录项目当前进度、阶段结论、阻塞项和交接信息。  
本文档不承载规范定义；涉及需求、功能、架构、数据库、UI 规范时，统一以 `docs/` 下对应规范文档为准。

## 一、文档职责

### 稳定基线文档

- `docs/开发指南.md`：长期开发规范、目录约束与交付要求，原则上只在开发规则变化时修改
- `docs/功能说明.md`：产品功能范围与功能边界，原则上只在功能范围变化时修改
- `docs/视觉风格与页面原型设计规范.md`：视觉语言与页面结构规范，原则上只在设计方向变化时修改
- `docs/需求分析.md`：产品目标、目标用户、核心场景与需求边界，原则上只在产品方向变化时修改

### 持续更新文档

- `docs/数据库架构.md`：当前数据存储事实、表结构与同步边界，数据结构变化时更新
- `docs/项目代码目录结构.md`：当前仓库目录事实与模块边界，结构变化时更新
- `docs/饮食管理.md`：饮食推荐与食物营养信息能力规划，饮食能力方案变化时更新
- `docs/项目状态.md`：阶段进度、结论、阻塞与交接信息，作为持续更新主文档
- `docs/文档索引.md`：文档职责、查阅顺序与维护约定，文档体系调整时更新

## 二、当前状态

- 项目当前处于移动端 PWA MVP 阶段
- 主流程已经形成可用闭环：注册 / 登录 -> 生成计划 -> 开始训练 -> 完成训练 -> 写入记录 -> 查看热图
- 文档体系已完成一轮收口，规范类内容已从进度类内容中分离
- “训练记录 -> 计划历史”的联动增强已完成首版落地，当前可从记录详情回看关联计划，并在计划历史页承接定位与使用痕迹
- 训练闭环承接体验已继续增强，记录页现在会对“刚完成训练”的结果提供顶部反馈、一键查看入口和详情高亮
- P0 主链路已完成一轮联调收口，计划同步、训练完成跳转、训练记录同步和记录页承接已验证通过
- 训练记录管理已继续补齐：当天详情可编辑/删除单条记录，并保留与计划历史的联动回看
- 饮食管理 P1 首版已落地：独立饮食页、静态知识检索、LLM 推荐生成和食物营养卡片已串通
- 饮食管理输入语义已从“忌口或限制”调整为“补充要求或问题”，当前既支持显式饮食限制，也支持自然语言补充需求
- 饮食推荐失败兜底已补上：上游模型异常时会自动重试，并回退到基于知识库的稳定推荐结果
- 饮食结果结构已继续增强：当前会对“补充要求或问题”输出显式回答区块，并把餐次建议拆成标题、食物、份量、原因和可替代项

## 三、当前已完成

- 账号体系：
  - 注册
  - 登录
  - 当前用户信息获取
  - 登录态恢复与路由守卫
- 训练计划：
  - 稳定通道生成
  - 流式通道生成
  - DeepSeek 失败时模板回退
  - 最近计划保存、恢复、删除
  - 当前计划轻量编辑（标题、总时长、动作顺序、删除动作）
  - 当前计划单动作替换（从动作库回填到编辑草稿）
  - 当前计划新增动作（从动作库追加到编辑草稿末尾）
  - 当前计划保存结果摘要（保存后反馈标题、总时长、动作替换/新增/删除/顺序变化）
  - 当前计划最近一次编辑结果卡片（计划页内保留最近一次保存摘要，支持再次回看）
  - 当前计划未保存变更提示（取消编辑、切页离开、关闭页面前提醒）
  - 计划历史列表独立页面
  - 计划云同步
- 训练闭环：
  - 从计划页进入训练会话
  - 按动作顺序推进训练
  - 完成训练后自动写入训练记录
- 训练记录：
  - 最近六周热图
  - 周/月切换
  - 训练统计
  - 趋势摘要
  - 当天详情增强展示
  - 单条训练记录编辑（时长、完成状态）
  - 单条训练记录删除
  - 记录详情关联计划摘要与回看入口
  - 手动记录 / 原计划已删除 / 可回看计划 三类详情状态区分
- 计划历史增强：
  - 按训练记录传入的 `planId` 自动展开并定位目标计划
  - 显示计划是否已用于训练及最近使用标识
  - 按“最近使用优先，其次创建时间倒序”排序历史计划
  - 支持“全部计划 / 已用于训练 / 未用于训练”前端筛选
- 动作库：
  - 静态动作数据读取
  - 关键词、部位、难度、器械筛选
  - 动作详情弹层
  - 计划页联动预筛选
  - 收藏与最近查看
- 饮食管理：
  - 独立饮食推荐页
  - 基于静态知识库的规则检索
  - LLM 生成 1 天饮食建议
  - 推荐结果内食物营养卡片展示
  - 首页饮食管理入口
  - 支持“补充要求或问题”输入，并对显式忌口继续做过滤处理
  - 上游 LLM 失败时自动重试一次，并可回退到稳定推荐结果
  - 对“补充要求或问题”输出显式回答区块
  - 餐次结果升级为结构化字段（标题、推荐食物、建议份量、搭配原因、可替代食物）
  - 已补充目标差异、工作日省时、外卖场景、乳糖不耐受和不吃辣等知识条目
