# FitMirror 项目状态

文档版本：v1.24  
最后更新：2026-04-03

本文档用于记录项目当前进度、阶段结论、阻塞项和交接信息。  
本文档不承载规范定义；涉及需求、功能、架构、数据库、UI 规范时，统一以 `docs/` 下对应规范文档为准。

## 一、文档职责

### 稳定基线文档

- `docs/开发指南.md`：长期开发规范、目录约束与交付要求，原则上只在开发规则变化时修改
- `docs/功能说明.md`：产品功能范围与功能边界，原则上只在功能范围变化时修改
- `docs/视觉风格与页面原型设计规范.md`：视觉语言与页面结构规范，原则上只在设计方向变化时修改
- `docs/需求分析.md`：产品目标、目标用户、核心场景与需求边界，原则上只在产品方向变化时修改

### 持续更新文档

- `docs/数据库架构.md`：当前数据存储事实、表结构与同步边界，数据结构变化时更新
- `docs/项目代码目录结构.md`：当前仓库目录事实与模块边界，结构变化时更新
- `docs/项目状态.md`：阶段进度、结论、阻塞与交接信息，作为持续更新主文档
- `docs/文档索引.md`：文档职责、查阅顺序与维护约定，文档体系调整时更新

## 二、当前状态

- 项目当前处于移动端 PWA MVP 阶段
- 主流程已经形成可用闭环：注册 / 登录 -> 生成计划 -> 开始训练 -> 完成训练 -> 写入记录 -> 查看热图
- 文档体系已完成一轮收口，规范类内容已从进度类内容中分离
- “训练记录 -> 计划历史”的联动增强已完成首版落地，当前可从记录详情回看关联计划，并在计划历史页承接定位与使用痕迹
- 训练闭环承接体验已继续增强，记录页现在会对“刚完成训练”的结果提供顶部反馈、一键查看入口和详情高亮
- P0 主链路已完成一轮联调收口，计划同步、训练完成跳转、训练记录同步和记录页承接已验证通过
- 训练记录管理已继续补齐：当天详情可编辑/删除单条记录，并保留与计划历史的联动回看

## 三、当前已完成

- 账号体系：
  - 注册
  - 登录
  - 当前用户信息获取
  - 登录态恢复与路由守卫
- 训练计划：
  - 稳定通道生成
  - 流式通道生成
  - DeepSeek 失败时模板回退
  - 最近计划保存、恢复、删除
  - 当前计划轻量编辑（标题、总时长、动作顺序、删除动作）
  - 当前计划单动作替换（从动作库回填到编辑草稿）
  - 当前计划新增动作（从动作库追加到编辑草稿末尾）
  - 当前计划保存结果摘要（保存后反馈标题、总时长、动作替换/新增/删除/顺序变化）
  - 当前计划最近一次编辑结果卡片（计划页内保留最近一次保存摘要，支持再次回看）
  - 当前计划未保存变更提示（取消编辑、切页离开、关闭页面前提醒）
  - 计划历史列表独立页面
  - 计划云同步
- 训练闭环：
  - 从计划页进入训练会话
  - 按动作顺序推进训练
  - 完成训练后自动写入训练记录
- 训练记录：
  - 最近六周热图
  - 周/月切换
  - 训练统计
  - 趋势摘要
  - 当天详情增强展示
  - 单条训练记录编辑（时长、完成状态）
  - 单条训练记录删除
  - 记录详情关联计划摘要与回看入口
  - 手动记录 / 原计划已删除 / 可回看计划 三类详情状态区分
- 计划历史增强：
  - 按训练记录传入的 `planId` 自动展开并定位目标计划
  - 显示计划是否已用于训练及最近使用标识
  - 按“最近使用优先，其次创建时间倒序”排序历史计划
  - 支持“全部计划 / 已用于训练 / 未用于训练”前端筛选
- 动作库：
  - 静态动作数据读取
  - 关键词、部位、难度、器械筛选
  - 动作详情弹层
  - 计划页联动预筛选
  - 收藏与最近查看

## 四、当前技术边界

- 后端当前承接账号体系、计划生成和本地数据同步
- 训练计划与训练记录事实源当前仍以前端 Dexie 为先，并同步到后端 MySQL
- 后端数据库当前包含账号、计划、训练记录模型
- 当前不把桌面端作为正式适配目标

## 六、下次继续时建议先做的事

1. 若继续优化计划生成体验，建议下一步优先把“生成中的主卡片内容”也做成更明确的占位态，避免旧计划内容在新一轮生成时继续占据主要视觉焦点
2. 若继续收口页面导航，建议检查登录/注册等非主流程页面是否也要统一到同一套顶部返回样式

## 七、本次更新记录

- 首页输入训练目标后点击“AI 生成”现在会直接跳转并自动开始生成，生成页无需再手动点第二次“生成计划”
- 计划生成页新增轻量生成会话持久化；切到其他页面再返回时，会继续展示生成状态，并在会话完成后恢复最新计划结果
- 计划生成页、历史计划页、训练记录页、训练页和动作库页的返回入口已统一上移到页面顶部；训练记录页的“手动补录”入口与对应逻辑已移除
- 本次已完成前端 lint、前端测试和 Chrome DevTools 实测；受本地后端未启动影响，浏览器控制台仍有既有 `ERR_CONNECTION_REFUSED` 报错，本轮前端主流程验证通过时采用了浏览器内注入登录态与模拟计划流式响应的方式完成交互核验

## 八、下一步建议

1. 下一步建议继续优化生成页在“新一轮生成进行中”时的内容切换策略，例如弱化旧计划卡片或切换为更明确的处理中占位
2. 若后续要继续收口训练记录体验，可补一轮“编辑后删除 / 多条同日记录 / 计划删除后回看记录”的浏览器回归
