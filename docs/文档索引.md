# FitMirror 文档索引

文档版本：v1.0  
最后更新：2026-04-02

本文档用于说明 `docs/` 目录中各文档的职责、更新频率与推荐查阅顺序。  
若文档内容与代码不一致，先以代码当前事实为准，再回写对应文档。

## 一、推荐查阅顺序

1. `docs/文档索引.md`：先理解文档分工与查阅入口
2. `docs/项目状态.md`：查看当前阶段进度、结论、阻塞与交接信息
3. `docs/需求分析.md`：查看产品目标、目标用户、场景与需求边界
4. `docs/功能说明.md`：查看当前产品功能范围与边界
5. `docs/开发指南.md`：查看开发规范、目录约束与交付要求
6. `docs/视觉风格与页面原型设计规范.md`：查看视觉语言与页面结构规范
7. `docs/数据库架构.md`：查看当前数据结构、表字段与同步边界
8. `docs/项目代码目录结构.md`：查看当前仓库结构与模块分布

## 二、稳定基线文档

以下文档原则上不承载阶段进度与临时记录，只在基线本身变化时修改；若需修改，应先确认修改理由成立，并明确说明修改范围。

- `docs/开发指南.md`：长期开发规范、目录约束、实现要求与交付要求
- `docs/功能说明.md`：产品功能范围、模块能力与功能边界
- `docs/视觉风格与页面原型设计规范.md`：视觉风格、页面结构、组件视觉约束与动效规范
- `docs/需求分析.md`：产品目标、目标用户、核心场景、需求边界与验收原则

## 三、持续更新文档

以下文档用于记录当前事实或阶段状态，后续需要持续维护：

- `docs/项目状态.md`：当前阶段进度、结论、阻塞、交接约定与下一步建议
- `docs/数据库架构.md`：Prisma 模型、Dexie 表结构、同步边界与数据存储事实
- `docs/项目代码目录结构.md`：当前仓库目录事实、模块边界与文件分布
- `docs/文档索引.md`：文档职责、查阅顺序与维护约定

## 四、维护规则

- 功能范围变化时，更新 `docs/功能说明.md`
- 产品目标、场景或需求边界变化时，更新 `docs/需求分析.md`
- 开发规则或目录约束变化时，更新 `docs/开发指南.md`
- 视觉方向或页面结构规范变化时，更新 `docs/视觉风格与页面原型设计规范.md`
- 数据模型、同步结构或表字段变化时，更新 `docs/数据库架构.md`
- 仓库目录或模块边界变化时，更新 `docs/项目代码目录结构.md`
- 阶段进度、阻塞、结论或交接信息变化时，更新 `docs/项目状态.md`

## 五、当前文档约定

- 当前 `docs/` 目录以这 8 份主文档为准
- 已移除的旧文档不再作为当前项目判断依据，也不应继续在 README 或其他文档中引用
