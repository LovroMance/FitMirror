# FitMirror AI 需求分析（移动端PWA专用）
文档版本:v1.2
最后更新：2026年3月
项目阶段：MVP

## 术语解释
- **MVP**：最小可用产品，仅实现核心功能，快速验证市场需求。
- **AI动作识别**：通过摄像头+AI模型（如PoseNet）判断用户动作是否标准。
- **Prompt工程**：通过设计高质量提示词模板，提升AI计划生成的相关性与个性化。
- **流式输出**：AI计划生成、训练反馈等支持分步/实时展示，提升交互体验。
- **RAG**：检索增强生成，用于饮食推荐等场景。
- **Web Speech API**：浏览器端语音播报训练提示。
- **静态JSON+GitHub数据源**：动作库等数据采用静态文件+GitHub托管，便于社区共建与热更新。
- **PWA能力**：manifest.json、service worker、推送、离线、添加到主屏幕等，全部面向移动端。

## 一、项目背景与市场分析（移动端PWA专用）

### 1.1 行业趋势

- 居家健身常态化：60%用户采用居家+健身房混合训练
- AI健身普及：AI教练成为标配
- 硬件轻量化：手机摄像头即可动作识别
- 移动端优先：用户主要通过手机进行健身训练，PWA成为主流形态

### 1.2 用户痛点与机会

| 用户类型 | 主要痛点       | 现有方案      | 缺口/机会点        |
| -------- | -------------- | ------------- | ------------------ |
| 健身小白 | 不会练、怕受伤 | 视频跟练      | 无人纠正，易练错   |
| 上班族   | 没时间、难坚持 | 健身App       | 计划死板，缺个性化 |
| 预算有限 | 请不起私教     | 免费内容+自学 | 无反馈，难坚持     |

> **机会点**：AI+摄像头+个性化+PWA，让每个人都能低成本获得专属AI私教，随时随地移动端体验。

## 二、产品定位与价值主张（移动端PWA专用）

### 2.1 一句话定位

你的AI私教，一面会说话的智能镜子（移动端PWA）。

### 2.2 价值主张

| 维度   | 价值主张         | 用户语言                 |
| ------ | ---------------- | ------------------------ |
| 核心   | AI生成个性化计划 | “说句话就有专属计划”     |
| 差异化 | 实时动作纠正     | “像镜子一样照出你的错误” |
| 情感   | 陪伴式鼓励       | “比私教更懂你的AI伙伴”   |

### 2.3 产品三要素

FitMirror = AI教练（个性化计划） + 动作镜（实时反馈） + 健身日记（可视化记录）

### 2.4 竞品与差异化

| 竞品 | 优势 | 劣势 | FitMirror差异化 |
| Keep | 内容多、社区强 | 无实时纠正 | AI生成+实时反馈 |
| Apple Fitness+ | 教练专业 | 需Apple设备、贵 | 跨平台、免费基础版 |
| 抖音跟练视频 | 免费、选择多 | 无互动、易练错 | AI纠正、个性化推荐 |
| 线下私教 | 专业、反馈快 | 贵、需预约 | 7x24小时、低成本 |

### 2.5 核心差异点

- **AI原生**：从0设计为AI-first
- **视觉驱动**：摄像头为核心交互
- **渐进式智能**：越用越懂你

## 七、隐私与伦理考量

### 7.1 数据隐私

- 摄像头数据仅本地识别（TensorFlow.js），不上传服务器
- 训练记录默认本地，可选云同步
- 明确告知数据处理方式，获取授权

### 7.2 安全边界

- 动作建议仅供参考，有不适请停止
- 用户自行承担运动风险
- 建议16岁以上使用

---

- **P0为必备功能**：上线必须具备AI动作教练（计划生成）、训练记录日历（GitHub热图风格）、动作库三大核心。
- **P1为进阶体验**：AI动作识别、饮食推荐、社交激励等增强用户粘性。
- **P2为可选增强**：语音播报、数据导出等提升产品亮点。
- 所有AI推理与绝大多数数据均本地完成，保障隐私。
- 本地数据存储采用IndexedDB，训练计划、记录、动作库等均存于本地。
- BFF（Backend For Frontend）架构：前端通过专用BFF服务获取必要的远程数据，BFF仅做数据聚合与转发，不存储用户隐私数据。
- 云端功能后续考虑。
