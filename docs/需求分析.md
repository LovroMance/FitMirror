# FitMirror 需求分析

文档版本：v4.0  
最后更新：2026-03-31

## 一、产品定位

FitMirror 是一个仅面向移动端 PWA 的轻量健身产品，核心目标是让用户完成一条最短可用训练闭环：

1. 注册并登录
2. 输入训练目标
3. 生成训练计划
4. 开始并完成训练
5. 自动写入训练记录
6. 在记录页回看结果

产品不定义为“全功能 AI 健身平台”，而定义为“能帮助用户完成一次训练并留下记录的移动端 MVP”。

## 二、目标用户与核心场景

### 目标用户

- 想快速开始训练的新手用户
- 需要轻量训练建议的居家训练用户
- 希望看到训练记录和连续打卡反馈的用户

### 核心场景

#### 场景 1：首次完成训练

1. 用户注册并登录
2. 用户输入训练目标
3. 系统生成训练计划
4. 用户从计划页进入训练会话
5. 用户完成训练
6. 系统自动生成训练记录

#### 场景 2：回看训练结果

1. 用户进入训练记录页
2. 查看最近训练热图和统计
3. 点击某一天查看训练详情

#### 场景 3：借助动作库理解计划内容

1. 用户在计划页点击动作名
2. 跳转到动作库并带入预筛选
3. 查看动作说明和注意事项

## 三、产品范围

### 范围内

- 账号注册、登录、登录态恢复
- 基于目标文本的训练计划生成
- 最近计划保存与恢复
- 训练会话推进与完成
- 本地训练记录写入与展示
- 动作库筛选与详情查看

### 范围外

- 动作识别与姿态纠正
- 饮食推荐
- 社交能力
- 云端训练记录同步
- 完整离线缓存与推送
- 多端适配和桌面端体验

## 四、业务约束

- 训练记录事实源为前端 Dexie，不新增后端训练记录持久化
- 后端只承接账号体系与计划生成能力
- 产品只支持移动端 PWA，不兼容桌面端为正式目标
- 设计语言以既有视觉规范为准，不开启脱离现状的大规模重设计

## 五、验收标准

### 功能验收

- 受保护页面在未登录时会跳转登录页
- 计划生成页可生成并展示训练计划
- 计划页存在开始训练入口
- 完成训练后会写入 `workout_records`
- 记录页可查看热图与当天详情
- 计划页可跳转动作库并带入预筛选

### 质量验收

- 根目录 `npm run lint` 通过
- `npm run test:frontend` 通过
- `npm run test:backend` 通过

### 文档验收

- 文档描述的接口、页面、数据结构可在仓库中定位
- 各文档之间不存在已实现状态冲突
