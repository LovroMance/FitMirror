# FitMirror 数据库架构

文档版本：v2.1  
最后更新：2026-04-01

本文档只描述当前数据存储事实，不承载未来扩展规划。  
当 Prisma 模型、Dexie 表结构或同步接口发生变化时，需要持续更新本文档。

## 一、数据存储原则

- 后端数据库使用 MySQL
- 后端数据库当前存账号、训练计划、训练记录
- 前端 Dexie 作为本地持久化层与页面事实读取层
- 动作库当前来自前端静态 JSON 文件

## 二、后端数据库结构

当前 Prisma 定义 `User`、`Plan`、`WorkoutRecord` 三个模型。

### `User`

| 字段名         | 类型       | 约束         |
| -------------- | ---------- | ------------ |
| `id`           | `Int`      | 主键，自增   |
| `email`        | `String`   | 唯一         |
| `username`     | `String`   | 唯一         |
| `passwordHash` | `String`   | 必填         |
| `createdAt`    | `DateTime` | 默认当前时间 |
| `updatedAt`    | `DateTime` | 自动更新时间 |

### `Plan`

| 字段名         | 类型       | 约束                 |
| -------------- | ---------- | -------------------- |
| `id`           | `Int`      | 主键，自增           |
| `userId`       | `Int`      | 关联 `User.id`       |
| `clientPlanId` | `String`   | 与 `userId` 联合唯一 |
| `goalText`     | `String`   | 必填                 |
| `planJson`     | `Json`     | 必填                 |
| `createdAt`    | `DateTime` | 必填                 |
| `updatedAt`    | `DateTime` | 自动更新时间         |

### `WorkoutRecord`

| 字段名           | 类型       | 约束                 |
| ---------------- | ---------- | -------------------- |
| `id`             | `Int`      | 主键，自增           |
| `userId`         | `Int`      | 关联 `User.id`       |
| `clientRecordId` | `String`   | 与 `userId` 联合唯一 |
| `date`           | `String`   | 必填                 |
| `duration`       | `Int`      | 必填                 |
| `completed`      | `Boolean`  | 必填                 |
| `planId`         | `Int?`     | 可空                 |
| `createdAt`      | `DateTime` | 默认当前时间         |
| `updatedAt`      | `DateTime` | 自动更新时间         |

## 三、前端本地数据结构

### `plans`

| 字段名      | 说明          |
| ----------- | ------------- |
| `id`           | 主键               |
| `userId`       | 用户 ID            |
| `clientPlanId` | 客户端计划唯一标识 |
| `goalText`     | 原始目标文本       |
| `planJson`     | 训练计划 JSON      |
| `createdAt`    | 创建时间           |
| `updatedAt`    | 更新时间           |

### `workout_records`

| 字段名      | 说明              |
| ----------- | ----------------- |
| `id`             | 主键               |
| `userId`         | 用户 ID            |
| `clientRecordId` | 客户端记录唯一标识 |
| `date`           | 训练日期           |
| `duration`       | 训练时长           |
| `completed`      | 是否完成           |
| `planId`         | 关联计划 ID，可选  |
| `createdAt`      | 创建时间           |
| `updatedAt`      | 更新时间           |

### `settings`

| 字段名      | 说明     |
| ----------- | -------- |
| `userId`    | 用户 ID  |
| `theme`     | 主题     |
| `unit`      | 单位制   |
| `updatedAt` | 更新时间 |

### `exercise_preferences`

| 字段名         | 说明               |
| -------------- | ------------------ |
| `id`           | 主键               |
| `userId`       | 用户 ID            |
| `exerciseId`   | 动作 ID            |
| `isFavorite`   | 是否收藏           |
| `lastViewedAt` | 最近查看时间，可选 |
| `updatedAt`    | 更新时间           |

## 四、当前数据边界

- 训练计划与训练记录存在“Dexie 本地事实层 + MySQL 云端同步存储”双写模型
- 同步接口已具备最小能力：`/api/plans/sync`、`/api/workout-records/sync`
- 后端不存动作库
- 动作库当前文件为 `frontend/public/data/exercises.json`
