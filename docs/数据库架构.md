# FitMirror 数据库架构

文档版本：v2.0  
最后更新：2026-03-31

本文档只描述当前数据存储事实，不承载未来扩展规划。

## 一、数据存储原则

- 后端数据库使用 MySQL
- 后端数据库当前只存账号体系数据
- 训练计划、训练记录和设置项当前存储在前端 Dexie
- 动作库当前来自前端静态 JSON 文件

## 二、后端数据库结构

当前 Prisma 仅定义一个 `User` 模型。

### `User`

| 字段名 | 类型 | 约束 |
| --- | --- | --- |
| `id` | `Int` | 主键，自增 |
| `email` | `String` | 唯一 |
| `username` | `String` | 唯一 |
| `passwordHash` | `String` | 必填 |
| `createdAt` | `DateTime` | 默认当前时间 |
| `updatedAt` | `DateTime` | 自动更新时间 |

## 三、前端本地数据结构

### `plans`

| 字段名 | 说明 |
| --- | --- |
| `id` | 主键 |
| `userId` | 用户 ID |
| `goalText` | 原始目标文本 |
| `planJson` | 训练计划 JSON |
| `createdAt` | 创建时间 |

### `workout_records`

| 字段名 | 说明 |
| --- | --- |
| `id` | 主键 |
| `userId` | 用户 ID |
| `date` | 训练日期 |
| `duration` | 训练时长 |
| `completed` | 是否完成 |
| `planId` | 关联计划 ID，可选 |

### `settings`

| 字段名 | 说明 |
| --- | --- |
| `userId` | 用户 ID |
| `theme` | 主题 |
| `unit` | 单位制 |
| `updatedAt` | 更新时间 |

## 四、当前数据边界

- 后端不存训练记录
- 后端不存训练计划
- 后端不存动作库
- 动作库当前文件为 `frontend/public/data/exercises.json`
