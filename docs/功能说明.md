# FitMirror 功能说明

文档版本：v2.0  
最后更新：2026-04-01

本文档描述当前产品能力和后续迭代边界，已按代码现状收口。

## 一、当前已实现能力

### 1. 账号体系

- 邮箱 + 用户名注册
- 邮箱或用户名登录
- 当前用户信息获取
- 本地 token 恢复登录态
- 路由守卫拦截未登录访问

### 2. 训练计划

- 输入目标生成个性化训练计划
- 支持流式计划生成状态反馈
- DeepSeek 失败时自动切换模板生成
- 最近一次计划自动恢复
- 历史计划列表回看、开始训练、复用到计划页、删除
- 计划历史云同步，支持跨会话恢复

### 3. 训练闭环

- 从计划页进入训练会话
- 从历史计划页进入训练会话
- 按动作顺序推进训练
- 完成训练后自动写入本地训练记录

### 4. 训练记录

- 近 6 周热图
- 近 30 天热图
- 训练天数、总时长、连续打卡统计
- 趋势摘要与最活跃训练日
- 点击当天热图查看当天详情
- 手动补录 10 / 20 / 30 分钟记录

### 5. 动作库

- 基于静态 `exercises.json` 读取动作数据
- 支持关键词、部位、难度、器械筛选
- 支持动作详情查看
- 支持计划页带关键字跳转预筛选
- 支持收藏与最近查看

## 二、当前技术边界

- 训练计划和训练记录仍保留前端 Dexie 缓存
- 训练计划和训练记录会同步到后端 MySQL
- 后端承接账号体系、计划生成和同步接口
- 当前不把桌面端作为正式适配目标
- 当前只有基础 `manifest.json`，不能视为完整 PWA 能力

## 三、当前未实现内容

### P1 方向

- 动作识别与实时纠错
- 智能饮食推荐
- 社交互动 / 点赞 / 排行榜

### P2 方向

- 语音播报
- 数据导出
- 完整 PWA 离线缓存、推送与添加到主屏幕体验
- 复杂计划协作或共享能力

## 四、核心数据流

1. 用户输入目标 -> 生成计划 -> 保存最近计划与历史计划
2. 用户从计划页或历史计划进入训练会话
3. 训练完成 -> 写入 `workout_records`
4. 热图、趋势摘要、当天详情读取本地记录并展示
5. 计划页动作名可跳转动作库并带入预筛选

## 五、当前阶段建议

- 优先继续稳住主链路回归与测试基线
- 后续二选一进入下一阶段：训练记录云同步或最小 PWA 能力
- 若功能发生变化，必须同步更新 `docs/项目状态.md` 和本文档
