# FitMirror 功能说明

更新时间：2026-03-31

本文档按当前代码状态整理，统一使用三态标记：

- `已实现`
- `部分实现`
- `未实现`

## 一、已实现

### 1. 账号与鉴权

- 注册、登录、`/me`
- JWT 持久化
- 登录态恢复
- 路由守卫

### 2. AI 计划生成

- `POST /api/plans/generate`
- `GET /api/plans/generate/stream`
- DeepSeek 优先、模板回退
- 前端流式状态反馈
- 计划生成结果来源标记
- 最近计划保存、恢复、删除

### 3. 真实训练闭环

- 计划页支持“开始训练”入口
- 训练会话页按当前计划动作顺序推进
- 完成训练后自动写入 `workout_records`
- 热图页可读取真实训练完成后的记录并展示到当天详情

### 4. 训练记录热图

- 最近六周热图展示
- 训练天数、总时长、连续打卡统计
- 点击日期查看当天详情

### 5. 动作库

- 静态数据源：`frontend/public/data/exercises.json`
- 关键词、部位、难度、器械筛选
- 动作详情弹层
- 计划页动作名跳转动作库预筛选
- 动作名归一化匹配，减少计划名与库名不一致导致的空结果

### 6. 本地数据层

- Dexie 三表：
  - `plans`
  - `workout_records`
  - `settings`
- `localStorage(fitmirror_latest_plan)` 一次性迁移到 Dexie

## 二、部分实现

### 1. 训练记录写入方式

- 真实训练完成后已可自动写入热图
- 热图页仍保留手动补录入口，作为补记和调试用途

### 2. PWA 能力

- 已有 `manifest.json`
- 尚未看到 service worker、离线缓存、推送通知等完整能力

### 3. 文档体系

- `当前进度与后续工作.md`、`协作进度记录.md` 已接近真实状态
- 部分早期产品文档刚开始收口，仍需持续同步

## 三、未实现

### 1. P0+ 增强

- 计划历史列表独立界面
- 热图周/月切换
- 趋势摘要卡
- 动作库收藏与最近查看

### 2. P1 / P2 规划能力

- AI 动作识别
- 饮食推荐
- 社交与排行榜
- 语音播报
- 数据导出
- 完整离线 PWA 能力

## 四、当前系统边界

- 本阶段不包含摄像头动作识别
- 训练记录仍主要存在前端本地
- 后端不承接训练记录持久化，只负责账号和计划生成能力
- 真实训练闭环以“计划驱动 + 自动写记录”为目标，不包含复杂训练计时、语音提示或姿态反馈
