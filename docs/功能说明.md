# FitMirror 功能说明

文档版本：v2.1  
最后更新：2026-04-01

本文档用于定义 FitMirror 的产品功能范围与功能边界。  
本文档只描述“产品提供什么能力”，不记录每轮开发进度、阶段结论、数据库字段、代码结构和视觉规范；对应内容分别查看 `docs/项目状态.md`、`docs/数据库架构.md`、`docs/项目代码目录结构.md`、`docs/视觉风格与页面原型设计规范.md`。

## 一、产品核心能力

FitMirror 当前围绕“生成训练计划 -> 完成一次训练 -> 留下训练记录 -> 回看结果”这一移动端健身闭环提供能力，核心功能包括：

- 账号注册、登录、登录态恢复与安全退出
- AI 训练计划生成与最近计划保存
- 历史计划回看、复用、删除与继续训练
- 训练会话执行与完成后自动写入训练记录
- 训练热图、趋势摘要、当天详情与手动补录
- 动作库检索、筛选、详情查看、收藏与最近查看
- 路由权限控制与基础交互反馈

## 二、功能模块说明

### 1. 账号体系

- 支持用户注册、登录、当前用户信息获取
- 支持登录态恢复与未登录路由拦截
- 支持安全退出并清理当前会话

### 2. 训练计划

- 用户输入训练目标后，可生成一份结构化训练计划
- 支持计划生成过程反馈与失败兜底
- 支持保存最近计划，供页面恢复与继续使用
- 支持历史计划列表的回看、复用、删除与直接开始训练
- 支持计划数据同步到后端

### 3. 训练会话

- 用户可从当前计划或历史计划进入训练会话
- 会话按动作顺序推进，并展示动作说明、组数、次数或时长信息
- 完成训练后自动生成训练记录，并回到记录查看链路

### 4. 训练记录

- 提供训练热图展示训练频率与活跃度
- 提供周期统计与趋势摘要
- 支持查看当天训练详情
- 支持手动补录训练记录
- 支持训练记录同步到后端

### 5. 动作库

- 基于静态动作数据提供动作浏览能力
- 支持关键词、部位、难度、器械等筛选
- 支持动作详情查看
- 支持动作收藏与最近查看
- 支持从训练计划联动跳转到动作库并带入筛选条件

### 6. 基础体验

- 提供移动端优先的页面导航与交互路径
- 提供表单校验、错误提示和关键操作反馈
- 提供路由权限控制与登录态拦截

## 三、当前功能边界

以下内容当前不属于 FitMirror 的产品基线能力，不在本文档中作为当前已提供功能维护：

- AI 动作识别与摄像头姿态纠正
- 智能饮食推荐
- 社交互动、点赞、排行榜
- 语音播报
- 训练数据导出
- 完整 PWA 离线缓存、推送与安装增强

## 四、维护原则

- 当产品功能范围发生变化时，更新本文档
- 当只是实现方式、目录结构、数据字段或阶段进度变化时，不在本文档中记录
- 若代码与本文档冲突，以代码当前事实为准，并同步修正文档
