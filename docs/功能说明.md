### P0（MVP核心，必须实现，移动端PWA专用）

- AI动作教练（生成个性化训练计划）
  - 计划生成提示词模板（Prompt Engineering）
  - 支持流式输出（如计划生成进度条/分步展示）
- 训练记录日历（GitHub活跃热图风格）
  - 以热力格子图展示每日训练活跃度，类似GitHub贡献热图
  - 训练详情页：动作列表+开始训练入口
- 动作库
  - 静态JSON文件存储动作数据
  - GitHub作为数据源，便于社区共建
  - 图表库（ECharts/Chart.js）用于训练数据可视化

### P1（MVP进阶，增强体验，移动端PWA专用）

- AI动作识别（摄像头）
  - TensorFlow.js + PoseNet实现本地推理
  - 深蹲等典型动作检测逻辑（如膝盖角度判断）
  - 训练中页面：摄像头画面+实时动作反馈
- 智能饮食推荐
  - RAG（检索增强生成）结合用户目标推荐饮食方案
- 社交激励
  - WebSocket实现实时互动/点赞/排行榜

### P2（可选增强，锦上添花，移动端PWA专用）

- 语音播报
  - Web Speech API实现训练语音提示
- 数据导出
  - 支持训练数据导出为CSV/Excel/图片
  - 文件生成与下载
- PWA能力
  - manifest.json、service worker、推送、离线、添加到主屏幕等

## 二、细分功能与技术亮点（移动端PWA专用）

- 计划生成的提示词模板：
  - 预设多种健身目标、时长、器械条件，用户可自定义
  - 示例："我想瘦肚子，每天10分钟，无器械"

  - 展示动作列表、每个动作的说明与演示

  - 摄像头画面全屏显示，实时动作反馈
  - 进度条/计时器/下一个动作预告
- 动作库：
  - 静态JSON文件，便于维护与扩展
  - GitHub托管，支持社区贡献与版本管理
- 训练详情页：

## 三、数据流与交互说明

1. 用户输入目标→AI生成计划→展示计划详情→开始训练
2. 训练中摄像头采集→PoseNet推理→实时反馈→训练完成
3. 训练结果写入本地→日历/数据分析可视化
4. 动作库/饮食库数据通过GitHub拉取，支持热更新

## 四、MVP范围说明

- 仅P0功能为上线必备，P1/P2为后续迭代方向
- 所有AI推理与绝大多数数据均本地完成，保障隐私
- 本地数据存储采用Dexie.js（IndexedDB 封装库），训练计划、记录、动作库等均存于本地
- 涉及用户隐私的健身数据、AI分析结果等全部前端本地存储，只有用户本人可见
- 后端仅存储账号、登录凭证、基础配置等必要信息，不存储训练计划、训练记录、动作分析等隐私数据
- BFF（Backend For Frontend）架构：前端通过专用BFF服务获取必要的远程数据，BFF仅做数据聚合与转发，不存储用户隐私数据
- 云端功能后续考虑

## 六、推荐工具与代码规范

- 日期处理推荐 day.js（轻量、链式API）
- 强制使用 ESLint、Prettier、Stylelint 保证代码风格统一，建议在CI中集成自动检查
- Git 提交校验推荐 Husky（Git 钩子），配合 lint-staged 实现提交前自动格式化和校验
- BFF（Backend For Frontend）架构：前端通过专用BFF服务获取必要的远程数据，BFF仅做数据聚合与转发，不存储用户隐私数据
- 云端功能后续考虑

## 六、基础功能补充（MVP必备）

### 1. 用户注册与登录

- 支持邮箱/用户名注册、登录
- 密码加密存储，支持找回密码（如邮箱找回/验证码）
- 第三方登录（如微信、GitHub等，后续可选）

### 2. 用户登出

- 一键安全退出，清除本地会话信息

### 3. 个人信息管理

- 头像上传/更换
- 昵称、邮箱等基础信息修改
- 账号安全设置（如修改密码）

### 4. 偏好设置

- 主题色/暗黑模式切换
- 训练目标、单位、通知等个性化配置

### 5. 错误与异常处理

- 全局错误提示与反馈（如登录失败、网络异常等）
- 表单校验与友好提示

### 6. 权限与安全

- 路由权限控制（未登录不可访问训练、数据等页面）
- 前端本地存储敏感数据加密（如Token）

### 7. 其他基础功能

- 响应式适配（电脑/移动端/PWA）
- 加载动画、骨架屏等提升用户体验
- 404/500等通用错误页
