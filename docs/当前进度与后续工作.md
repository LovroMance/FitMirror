# FitMirror 当前进度与后续工作

更新时间：2026-03-31

## 1. 当前总体判断

项目 P0 主闭环已经稳定，且“生成计划 -> 开始训练 -> 完成训练 -> 自动写记录 -> 热图展示”已经打通。当前阶段重点从补主链路，转为继续做现状文档基线与后续能力规划：

- 文档和代码状态对齐
- 巩固当前测试与回归基线
- 规划训练记录的可追溯性与同步方案
- 评估下一阶段功能优先级

## 2. 当前已完成能力

### 账号与鉴权

- 注册、登录、`/me`
- JWT 持久化
- 登录态恢复
- 路由守卫

### 计划生成

- `POST /api/plans/generate`
- `GET /api/plans/generate/stream`
- DeepSeek 优先、模板回退
- 前端流式状态反馈与来源标签

### 本地数据层

- Dexie 三表：
  - `plans`
  - `workout_records`
  - `settings`
- 最近计划保存、恢复、删除
- `localStorage(fitmirror_latest_plan)` 向 Dexie 的一次性迁移

### 训练记录热图

- 最近六周热图展示
- 训练天数、总时长、连续打卡统计
- 点击日期查看详情
- 训练完成后自动写入真实训练记录
- 手动补录 10/20/30 分钟入口保留，作为补记和调试入口

### 训练会话闭环

- 计划页支持“开始训练”
- 新增训练会话页 `/workouts/session`
- 按当前计划动作顺序推进训练
- 完成训练后自动写入 `workout_records`

### 动作库

- 静态数据源：`frontend/public/data/exercises.json`
- 关键词、部位、难度、器械筛选
- 动作详情弹层
- 与计划页联动预筛选

## 3. 当前明确未完成或仅部分完成

### 功能层面

- 计划历史列表尚未提供独立界面
- 热图周/月切换、趋势摘要尚未实现
- 收藏、最近查看等动作库增强尚未实现
- 云端训练记录同步尚未实现

### 体验层面

- 登录/注册/动作库视觉统一仍在继续
- 主链路虽可用，但还缺一次完整收口回归

### 平台能力

- 当前只有 `manifest.json`
- service worker、离线缓存、推送通知尚未实现

### 后续版本功能

- AI 动作识别
- 饮食推荐
- 社交与排行榜

## 4. 当前主要问题

1. 当前训练记录事实源仍是前端 Dexie，本地闭环已通，但还没有云端同步。
2. Windows 下前端开发环境偶发 `vite spawn EPERM`，测试与开发服务需要按已验证方式启动。
3. PWA 当前只有 `manifest.json`，离线、service worker、推送仍未实现。
4. 工作区存在用户本地未提交改动，修改时需要避开覆盖。
