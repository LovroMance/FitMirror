# FitMirror 当前进度与后续工作

更新时间：2026-03-31

## 1. 当前总体判断

项目 P0 主闭环已经基本成型，当前阶段重点不再是继续铺新功能，而是做收口：

- 文档和代码状态对齐
- 补齐最小测试集
- 回填验收结果
- 完成认证页和动作库的视觉统一

## 2. 当前已完成能力

### 账号与鉴权

- 注册、登录、`/me`
- JWT 持久化
- 登录态恢复
- 路由守卫

### 计划生成

- `POST /api/plans/generate`
- `GET /api/plans/generate/stream`
- DeepSeek 优先、模板回退
- 前端流式状态反馈与来源标签

### 本地数据层

- Dexie 三表：
  - `plans`
  - `workout_records`
  - `settings`
- 最近计划保存、恢复、删除
- `localStorage(fitmirror_latest_plan)` 向 Dexie 的一次性迁移

### 训练记录热图

- 最近六周热图展示
- 训练天数、总时长、连续打卡统计
- 点击日期查看详情

说明：

- 当前记录写入仍通过“模拟完成 10/20/30 分钟”按钮进行。

### 动作库

- 静态数据源：`frontend/public/data/exercises.json`
- 关键词、部位、难度、器械筛选
- 动作详情弹层
- 与计划页联动预筛选

## 3. 当前明确未完成或仅部分完成

### 功能层面

- 真实训练闭环尚未接通记录写入
- 计划历史列表尚未提供独立界面
- 热图周/月切换、趋势摘要尚未实现
- 收藏、最近查看等动作库增强尚未实现

### 体验层面

- 登录/注册/动作库视觉统一仍在继续
- 主链路虽可用，但还缺一次完整收口回归

### 平台能力

- 当前只有 `manifest.json`
- service worker、离线缓存、推送通知尚未实现

### 后续版本功能

- AI 动作识别
- 饮食推荐
- 社交与排行榜

## 4. 当前主要问题

1. 真实训练闭环仍未接通记录写入，当前保留模拟写入入口。
2. Windows 下前端开发环境偶发 `vite spawn EPERM`，测试与开发服务需要按已验证方式启动。
3. PWA 当前只有 `manifest.json`，离线、service worker、推送仍未实现。
4. 工作区存在用户本地未提交改动，修改时需要避开覆盖。
